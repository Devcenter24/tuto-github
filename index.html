Avant de commencer

Le principe

GitHub peut heberger gratuitement des pages statiques. Cela veut dire qu'un site en HTML, CSS et JavaScript peut etre mis en ligne directement depuis un depot.

1

Prepare le dossier

Ton site doit contenir au minimum un fichier `index.html`. C'est la page d'accueil.

2

Cree un depot

Sur GitHub, cree un nouveau repository pour stocker les fichiers de ton site.

3

Envoie les fichiers

Utilise Git pour pousser ton dossier local sur GitHub depuis le terminal.

4

Active Pages

Dans `Settings > Pages`, choisis la branche `main` et le dossier racine.

Le parcours complet

Etape par etape

Voici le chemin le plus simple si tu debutes et que tu veux publier un premier site statique sans framework.

Publication avec GitHub Pages

  1. Etape 01

    Creer les fichiers du site

    Commence avec `index.html`. Si tu veux, ajoute aussi `style.css` et un dossier `images`. Verifie que la page s'ouvre bien en local.

  2. Etape 02

    Initialiser Git

    Ouvre un terminal dans le dossier du site, lance `git init`, ajoute les fichiers, puis cree un premier commit.

  3. Etape 03

    Connecter GitHub

    Cree ton depot en ligne, copie l'URL et ajoute-la avec `git remote add origin ...`. Ensuite, pousse la branche `main`.

  4. Etape 04

    Activer la publication

    Va dans les parametres du depot puis dans `Pages`. Choisis `Deploy from a branch`, selectionne `main` et le dossier `/root`.

  5. Etape 05

    Recuperer le lien

    GitHub genere une adresse du type `https://ton-compte.github.io/nom-du-depot/`. Au bout de quelques instants, ton site devient accessible.

Conseils utiles

  • Le nom du fichier doit etre exactement `index.html` pour la page d'accueil.
  • Si les modifications ne s'affichent pas, attends une minute puis recharge la page.
  • Evite les espaces dans les noms de fichiers. Prefere `mon-site` ou `images/logo.png`.
  • Chaque changement local doit etre ajoute, committe puis pousse pour mettre le site a jour.

Le terminal

Les commandes a utiliser

Remplace l'URL par celle de ton propre depot GitHub. Tu peux copier ce bloc tel quel puis adapter le nom du repository.

Premier envoi du site

Utilise ces commandes si tu pars d'un dossier local deja pret.

git init
git add .
git commit -m "Ajout du site"
git branch -M main
git remote add origin https://github.com/ton-compte/ton-repo.git
git push -u origin main

Mise a jour du site

Quand tu modifies tes fichiers, il suffit ensuite de refaire ceci.

git add .
git commit -m "Mise a jour du site"
git push

Blocages courants

Erreurs frequentes

Si ton site ne sort pas comme prevu, regarde ici d'abord. Ce sont les problemes les plus classiques quand on publie son premier projet sur GitHub Pages.

404 au chargement

La page d'accueil n'est pas trouvee

GitHub Pages cherche surtout un fichier `index.html` a la racine du projet ou du dossier publie.

Correction: verifie le nom exact `index.html` et son emplacement.

Pages inactive

Le depot existe mais rien n'est en ligne

Pousser les fichiers ne suffit pas toujours. Il faut aussi activer GitHub Pages dans les parametres du repository.

Correction: ouvre `Settings > Pages`, choisis `main` puis `/root`.

Ancienne version

Les changements ne s'affichent pas

Soit le dernier `git push` n'a pas ete fait, soit le deploiement est encore en cours, soit le navigateur garde l'ancienne page en cache.

Correction: refais `git add`, `commit`, `push`, puis recharge la page.

Images cassees

Les fichiers ne se chargent pas

Les chemins peuvent etre faux, surtout si le nom du fichier change entre majuscules et minuscules ou si le dossier n'est pas au bon endroit.

Correction: garde des noms simples comme `images/logo.png`.

Questions frequentes

FAQ rapide

Quelques reponses simples aux blocages les plus courants quand on commence avec GitHub Pages.

Pourquoi mon site ne s'affiche pas tout de suite ?

La publication prend parfois quelques dizaines de secondes. Recharge la page un peu plus tard et verifie dans `Settings > Pages` que le deploiement est bien termine.

Faut-il payer pour publier un site simple ?

Non. GitHub Pages permet de publier gratuitement un site statique compose de fichiers HTML, CSS et JavaScript.

Puis-je utiliser mon propre nom de domaine ?

Oui. GitHub Pages permet aussi d'ajouter un domaine personnalise dans les parametres du depot.
